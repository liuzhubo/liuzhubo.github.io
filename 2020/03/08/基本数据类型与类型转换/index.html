# 数据类型与类型转换

# 数据类型

​ Js中有5种简单数据类型（也称为基本数据类型）：Undefined、Null、Boolean、Number和String。还有一种复杂数据类型——Object，Object本质上是由一组无序的键值对组成的。

​ Js是一门松散类型的语言，也称为弱类型语言，定义的变量可以用来保存任何类型的数值。因为没有强约束，在进行变量之间的运算时，很容易出现不同数据类型进行运算的情况，这时候类型转换就登场了。

​ js的数值转换分为强制转换与隐式转换。

# 强制转换

# Boolean()

将变量转化为布尔类型，当转换的值为“空字符串”、“数字0”、“undefined”，“null”时，返回false，否则返回true。

var b1 = Boolean("s");//返回true，非空字符串
var b2 = Boolean(0);//返回false，数字0
var b3 = Boolean(new Object());//返回true，对象
# Number()

将变量转化为数值类型，布尔值会被转化成0或1，null返回0，undefined返回NaN。

如果是字符串。遵循以下原则：

1、只包含数值，会被转化为十进制数值。

2、字符串为空，返回0。

3、字符串包含有效的十六进制格式，被转化为相同大小的十进制数值。

4、其余情况一律转化为NaN。

如果是对象，则调用对象的valueof（）方法，然后按照以上规则转换。

var n1 = Number(false); //0
var n2 = Number(undefined) //NaN
var n3 = Number(null) //0
var n4 = Number("1.2") //1.2
var n5 = Number("1.2.3") //NaN
var n6 = Number(new Object()) //NaN
# String()

将所有类型转化为字符串

# String()和toString()、Number和parseInt()

null和undefined没有toString方法，而String()适用于所有数据类型。

var t1 = null;
var t2 = String(t1);//t2的值 "null"
var t3 = t1.toString();//这里会报错
var t4;
var t5 = String(t4);//t5的值 "undefined"
var t6 = t4.toString();//这里会报错

Number()转化的整个值，而parseInt只转换变量开头的值。

var str = '1234aaa';
var num1 = Number(str);//NaN
var num2 = parseInt(str);//1234

# 隐式转换

	由于js是弱类型语言，当运算符在运算时，如果两边数据不统一，CPU就无法计算，这时编译器会自动将运算符两边的数据做一个数据类型转换，转成一样的数据类型再计算，这里对出现次数较多的情况进行阐述。

# 一元加减运算符

​ 在对非数值应用一元加减运算符时，该操作符会像Number()一样对值进行类型转换，所有的规则与Number保持一致。

var num1 = 1 + false;//1
var num2 = 1 + null; //1
var num3 = 1 + undefined; //NaN
var num4 = 1 +(new Object()); //NaN
var num5 = 1 + 'a';// '1a' ，此时的+其实为字符串操作符

# 布尔操作符

​ ！、&&、||分别为非、与、或。在对非布尔类型的数值进行运算时，会采用Boolean()方法，按照相应规则进行换算。

alert(!"blue") //false
alert(!NaN) // true
alert(!null) //true

# 关系运算符

< 、>、==当应用于非数值类型时，采用以下规则：

1、如果两个操作数都是字符串，则比较两个字符串首字母的字符编码值。

2、如果一个操作数为数值，则将另外一个操作数转化为数值，再进行比较。

3、如果一个操作数为布尔值，则先将其转化为数值，再进行比较。

"2">10 //false
"2">"10" //true
2>true //true

# 相等运算符

​ js中有相等运算符（==）和全等运算符（===），全等运算符只有在数据类型相同且数值相等的情况下才会返回true，故此处仅讨论相等运算符的类型转换。遵循以下原则：

1、如果一个操作数是布尔值，会将其转先转换为数值0或1。

2、如果一个操作数是字符串，另一个操作数是数值，会先将字符串转换为数值。

3、若一个操作数是对象，另一个不是，则会先调用对象的typeof()方法。

4、null和undefined是相等的。

5、两个对象是比较是否是同一个对象。

true == 1 //true
1 == '1' //true
null == undefined //true
var obj1 = {a:1};var obj2 = {a:1};
obj1 == obj2 //false,这两个变量不是指向同一个对象

# 总结

​ 以上即为js在类型转换时常用的情形，然而个人认为尽管js编译器会进行隐式的数据转换，在日常开发中还是尽量避免不同数据类型进行运算。而在今天，技术上已经可以对js的变量分配数据类型，如flow插件或typeScript，在日后的开发中可以尝试使用。
