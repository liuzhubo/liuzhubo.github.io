# 闭包与其特点的原理

# 闭包

​ 闭包是指有权访问另一个函数作用域中的变量的函数。在理解上，可以将其分为闭和包。闭就是函数外部无法访问到函数内部的变量对象，整个函数属于一个封闭的状态。包就是函数内部可以通过某种方式，将内部变量“包出去”，使函数外也可访问函数内部变量。

​ 创建闭包的常见方式，是在一个函数内部创建另一个函数，然后以return的方式暴露出去。

# 特点

# 在函数外部可访问到函数内部的变量

function outer() {
  const a = 1;
  return function inner() {
    console.log(a);
  }
};
const foo = outer();
foo();//1

​ 在例子中可以很清晰的看到，在全局环境下执行的函数foo，访问到了函数outer内部的变量a，将其打印出来。这与我们对函数的常规认知有很大的出入。所以闭包函数，就是通过特殊的方式，使外部函数拿到本不该拿到的变量值。

# 作用域链延长

​ 我们知道，当函数创建时，会在其内部的[[scope]]属性中，保存一个预先包含全局变量对象的作用域链。当函数执行时，会为函数创建一个执行环境，通过复制内部[[scope]]属性中的作用域链构建起当前环境的作用域链，然后再创建一个新的变量对象，推入当前作用域链的前端。而在函数执行完毕后，局部变量对象会被销毁，内存中仅保存全局作用域。

​ 而当通过在函数中return另一个函数的方式创建闭包函数inner时，此时inner函数中[[scope]]属性的作用域链包括全局变量对象和外部函数outer的变量对象，然后当函数执行时，创建当前执行环境的变量对象，推入作用域链，此时作用域链中有三个变量对象的引用：inner->outer->window，这被称之为作用域链延长。

# 同一个函数创建两个闭包函数的问题

function outer() {
  let i = 0;
  return function inner() {
    i++;
    console.log(i)
  }
};
const foo1 = outer();
const foo2 = outer();//此处相同的函数创建了两个闭包函数
foo1();//1
foo1();//2
foo1();//3
foo2();//1
foo2();//2
foo2();//3

​ 每次外部函数执行的时 候，外部函数的引用地址不同，都会重新创建一个新的地址。在上述例子中，foo1和foo2的引用地址不同，所以在两个函数的执行环境中创建了两个outer函数的变量对象。所以两个函数对i所做的操作互不干扰。而如果向上查找到的i是属于全局变量环境，两者将操作的是同一个地址中的值，因为全局变量对象一直存在且只有一个。

# 内存泄漏

​ 通常当函数执行完毕时，其活动对象会连同执行环境一起被销毁，内存中只会保存全局变量对象。而在闭包函数中，因为在创建时其内部的[[scope]]属性的作用域链保持了对外部函数的引用，所以外部函数的活动对象将一直存在于内存中。只要闭包函数一直不被销毁，内存中就会有一块被占用，这被称为内存泄漏。

​ 而要解决这个问题，在平常开发中，要及时解除闭包函数的引用。

function outer() {
  const a = 1;
  return function inner() {
    console.log(a);
  }
};
const foo = outer();
foo();//1
foo = null;//此时foo不再是一个闭包函数，其对外部函数的活动对象的引用已经被解除，垃圾回收机制将顺利回收内存。

# 总结

​ 闭包函数是指能够引用另一个函数活动对象的函数。在使用闭包函数时，要注意及时将其销毁，否则会发生内存泄漏的问题。
