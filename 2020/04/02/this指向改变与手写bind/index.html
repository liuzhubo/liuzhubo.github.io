# this指向改变与手写bind()

# this的指向

​ 在java中，this只指向类的实例对象，且不能改变。然后在javaScript中，this引用的是函数据以执行的环境对象。因为执行环境的多变，this的指向也会相应改变，并且非常灵活。我们先来讨论在不手动改变的情况下，this的指向问题。

当在全局作用域中调用函数时，this引用的是window

var a = 1;
function foo() {
  console.log(this.a); //1
  this.b = 2;
};
console.log(b); //2

​ 这个很好理解，这里需要注意的是，this是一个对象，所以是按引用传递，当在函数里改变this中的值时，会影响到全局环境的对象变量，所以尽量避免。

当由对象来调用函数时，this引用的是这个对象

var obj = {
  foo: function() {
    console.log(this.a);
    console.log(this.b);
  },
  a: 1
};
var a = 2;
var b = 3;
obj.foo(); //1，undefined

​ 这里同样很好理解，当foo()执行时，它的this指向了obj对象，所以a和b也是取对象中的值，当对象中未声明值时，打印出undefined而不是全局对象中的3。这里需要注意的是，一定要看清是对象引用的函数，否则会出现下列情况。

在对象的某一个属性中，执行函数，this引用的还是window

var foo = {console.log(this.a)};
var a = 1;
var obj = {
  a:2,
  fn:function() {
    foo();
  }
};
obj.fn();//1

​ 这里就不太好理解了，实际上当foo()执行时，执行环境栈中只有全局环境，所以this引用的还是window，如果在fn中直接打印this，这时候的this才应该是obj对象。

实例的this指向它自己

function Person() {this.get = function() {console.log(this.a);}};
var p = new Person();
p.a = 1;
p.get(); //1

箭头函数没有自己的this，只会继承函数上下文的this

​ 箭头函数是es6的新语法，这一点在vue与react中使用非常之多，值得关注一下。

# 手动改变this指向

​ call()、apply()、bind()三者为改变thid指向的方法。call()和apply()在效果上基本相同，不同点在于传入参数的格式不一样，且使用之后都会自动执行函数。bind()会返回一个新的函数，而且不会自动执行。

var obj1 = {
  a: 1,
  get: function(c,d) {
    console.log(this.a);
  }
};
var obj2 = {
  a: 2,
  get: function(c,d) {
    console.log(this.a);
  }
};
obj1.get.call(obj2, 3, 4); //2
obj1.get.apply(obj2, [3,4]);//2
var fn = obj1.get.bind(obj2, 3, 4);
fn();//2

​ 这里需要注意的是，bind()方法创建的一个新函数，与原来的函数并不在同一个内存空间中。

# 手写bind代码

​ 我们可以先手动实现一个call()方法

//手写call方法
Function.prototype.myCall = function(context=window, ...args) {
  context.foo = this;
  const result = context.foo(...args);
  delete context.foo;
  return result;
};
var foo = function (b){console.log(this.a+b)};
var obj = {a:1};
foo.myCall(obj, 1) //2

​ 而apply()方法的实现过程大同小异

//手写apply方法
Function.prototype.myApply = function(context = window, args = []) {
  if (!args instanceof Array) {
    //抛出错误
    return
  };
  context.foo = this;
  const result = context.foo(...args);
  delete context.foo;
  return result;
}

​ 这样便模拟了改变this指向，同时自动执行函数并返回结果;而bind和apply的区别在于,bind是返回一个绑定好的函数,apply是直接调用。并且要考虑到使用new去调用

///手写bind方法
Function.prototype.myBind = function(context, ...args) {
  const _this = this;
  let newFun = function() {
     return _this.call(this instanceof newFun ? this : context, ...args, ...arguments)
  };//这里判断新函数是作为构造函数使用，还是普通的函数，如果是作为构造函数，this引用的是实例
  newFun.prototype = this.prototype;//这里让新函数拥有与原函数相同的原型
  return newFun;
}

# 总结

​ this的指向看起来复杂，实际上只需要记住一点，谁调用this就指向谁。而apply、call、bind方法的手写，只要深刻理解了上述代码，也就不成问题了。
