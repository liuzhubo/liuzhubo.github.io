# 初级前端基本知识整理

​ 最近准备面试，这里将目前收集到的一部分初级前端工程师基本知识点做一个归纳整理，方便查阅与复习。

# HTML

# HTML语义化理解

​ HTML语义化简单简单来说就是使标签具有自己的含义，例如标题适用h1、段落使用p等。他的优点具有以下几点：

1、使整个页面结构更加清晰。

2、有利于SEO搜索优化。有助于爬虫抓取更多的信息。

3、便于团队开发与维护。

4、方便其他设备解析（如屏幕阅读器、盲人阅读器、移动设备）以意义的方式来渲染网页；

# HTML5新特性

1、新增了一些语义化标签：header、footer、nav、main等。

2、提供了音频和视频的标准，即audio、vedio。

3、canvas绘图。

4、SVG绘图。

5、web worker 创建一个独立的线程。

6、提供了localStorage和sessionStorage对象。

7、webSorket协议，一种客户端与服务端的通信机制。

# DOCTYPE有什么作用？标准模式与混杂模式如何区分？它们有何意义

​ 告诉浏览器使用哪个版本的HTML规范来渲染文档。DOCTYPE不存在或形式不正确会导致HTML文档以混杂模式呈现。标准模式（Standards mode）以浏览器支持的最高标准运行；混杂模式（Quirks mode）中页面是一种比较宽松的向后兼容的方式显示。

# 行内元素与块级元素的区别

1、行内元素与其他行内元素在一行，块级元素独占一行。

2、行内元素不可定义宽高，且margin和padding都是上下无效、左右有效。块级元素可以设置宽高与边距。

常见行内元素：a、span、img、input

常见块级元素：div、ul、li、p、h1

# 浏览器内核

浏览器内核分为渲染引擎和JS引擎。

渲染引擎负责取得网页的内容，JS引擎负责解释与执行javaScript。

常见浏览器的内核:Trident（IE、360浏览器）、webkit(safari)、blink(chrome);

# 元素的alt和title有什么异同？

在alt和title同时设置的时候，alt作为图片的替代文字出现，title是图片的解释文字。

# iframe框架

1、iframe能够原封不动的把嵌入的网页展现出来。

2、如果遇到加载缓慢的第三方内容如图标和广告，这些问题可以由iframe来解决。

# CSS

# 盒子模型

所有的HTML可以看成是一个盒子.分为标准盒子模型和怪异盒子模型。

标准盒子模型：box-sizing:content-box，width为content区域的宽度。

怪异盒子模型：box-sizing:border-box, width为content区域+内边距+边框。

# 清除浮动

清除浮动主要是为了解决，父元素因为子级元素浮动引起的内部高度为0的问题

/*在最后一个子级元素后添加额外的标签*/
.clear {
  clear:both;
}
/*给父级元素添加overflow:hidden，触发BFC*/
.father {
  overflow:hidden;
}
/*使用双伪元素*/
.clear:after,.clear:before {
  content:"";
  display:tabel;
}
.clear:after {
  clear:both;
}
.clear {
  *zoom:1;
}

# rem和em的区别

Rem相对于根元素html的font-size，em相对与父元素的font-size。

# BFC

块级格式化上下文，理解起来就是一个盒子，两个不同的盒子间布局互不干涉。

产生BFC的方法：

1、根元素。

2、float属性不为none。

3、position为absolute或fixed。

4、display为inline-block、flex。

5、overflow不为visible

# link和@import的区别

1、link引入的css会被同时加载，@import只有在页面加载完毕后才被加载。

2、@import只能引入css，link还能引入rss等
